# Eden Development Roadmap

## Phase 1: Launchable Eden

Focus:
- Publish a clear builder-to-consumer loop
- Keep service pricing visible
- Keep Eden Credits wallet behavior explicit
- Keep owner reconciliation and payout accounting inspectable

Current shape:
- Builders can create, price, publish, and preview services
- Consumers can discover services, top up credits, and run services
- Owners can inspect payments, payouts, and operational state

## Phase 2: Builder Automation

Focus:
- Expand AI-assisted service creation
- Improve builder-side workflow acceleration
- Add smarter launch-readiness guidance

Target outcomes:
- Faster service scaffolding
- Better publish preparation
- More reusable workspace intelligence

## Phase 3: Contribution Layer

Focus:
- Introduce structured contribution and extension paths
- Enable templates, reusable service patterns, and shared assets
- Support broader ecosystem participation without breaking the core loop

Target outcomes:
- Shared templates and building blocks
- More repeatable service creation flows
- Clear contribution boundaries for future collaborators and agents

## Phase 4: Autonomous Eden

Focus:
- Move from guided tools to agent-driven operations
- Support planned release orchestration and autonomous workflow execution
- Let Eden coordinate more of its own product, builder, and platform operations

Target outcomes:
- Agent-managed release planning
- Automated builder assistance across the workspace
- Autonomous platform optimization on top of stable guardrails
