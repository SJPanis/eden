# Release 002: Builder Tools

## Goal

Release 002 expands the builder tooling layer so Eden becomes faster to build with and easier to scale through assisted workflows.

## Planned Builder Tooling

### AI Service Scaffolding
- generate structured service drafts from short builder prompts
- scaffold name, description, category, tags, and pricing defaults
- connect generated output directly into editable workspace drafts

### Builder AI Assistant Improvements
- stronger publish-prep guidance
- richer reusable history and comparison of prior assistant runs
- better patch suggestions tied to launch readiness, pricing, and service quality

### Service Template Generation
- generate reusable service templates for common business patterns
- support faster setup for repeatable service types
- prepare a base for future shared contribution and template libraries

## Intended Outcomes
- reduce the time from idea to launch-ready service
- make service creation more repeatable
- improve builder confidence before publish
- prepare Eden for more automated release and workspace tooling later

## Dependency on Release 001

Release 002 builds on the launchable Eden base:
- the consumer discovery loop already exists
- pricing, wallet, and publish flows are already visible
- builder and owner analytics already provide the operating context

That allows builder tooling to focus on acceleration rather than basic platform assembly.
